Recursive Sequence Guide

A recursive sequence defines each new term by referring to one or more terms that came before it. This guide explains how to read recursive rules, generate terms, and connect recursive notation with familiar sequence formulas.

On this page: definition, notation, examples, recursive versus explicit rules, and FAQ.

What is a recursive sequence?

A recursive rule describes how to obtain a term from earlier terms. It must also provide enough initial information to begin the process. For example, the rule a(n) = a(n - 1) + 4 says to add 4 to the previous term, but it does not identify a unique sequence until a starting value is given.

With a(1) = 3, the rule produces 3, 7, 11, 15, and so on. With a(1) = 10, the same recursive step produces 10, 14, 18, 22. The initial condition is therefore part of the definition, not an optional detail.

How to read recursive notation

  • a(n) is the term at position n.
  • a(n - 1) is the term immediately before it.
  • a(n - 2) is the term two positions earlier.
  • An initial condition such as a(1) = 5 supplies a starting value.
  • A restriction such as n > 1 indicates when the recursive rule applies.

If a rule refers to the two previous terms, it normally needs two initial values. This is why the Fibonacci sequence begins with two stated terms.

Worked recursive sequence examples

Constant addition

Given a(1) = 6 and a(n) = a(n - 1) + 5, repeatedly add 5:

6, 11, 16, 21, 26

This is also an arithmetic sequence with common difference 5.

Constant multiplication

Given a(1) = 2 and a(n) = 3a(n - 1), repeatedly multiply by 3:

2, 6, 18, 54, 162

This is also a geometric sequence with common ratio 3.

Using two earlier terms

Given a(1) = 1, a(2) = 3, and a(n) = a(n - 1) + a(n - 2):

1, 3, 4, 7, 11, 18

Each new term is the sum of the two terms before it.

A changing operation

Given a(1) = 2 and a(n) = a(n - 1) + n:

2, 4, 7, 11, 16

The amount added changes with n, so this sequence is not arithmetic.

Recursive rules versus explicit formulas

A recursive rule builds a term from earlier terms. An explicit formula calculates a term directly from its position. For the arithmetic sequence 3, 7, 11, 15, the recursive rule is a(1) = 3 and a(n) = a(n - 1) + 4. Its explicit formula is a(n) = 3 + 4(n - 1).

Recursive descriptions often make the pattern easier to see. Explicit formulas are usually faster when you need a distant term because they avoid calculating every preceding term. Some sequences have simple versions of both; others are much more naturally described recursively.

Common mistakes

  • Leaving out the initial value or values.
  • Using the rule at a position where the required earlier term does not exist.
  • Confusing a(n - 1) with a(n) - 1.
  • Changing the previous term mentally instead of substituting it carefully.
  • Assuming every recursive sequence is arithmetic or geometric.

Recursive sequence FAQ

Does every recursive sequence need a starting value?

Yes. It needs enough initial values to calculate the first term covered by the recursive rule.

Can a recursive rule use more than one earlier term?

Yes. Fibonacci-type rules use two earlier terms, and more advanced recurrences can use several.

Is a recursive sequence the same as an arithmetic sequence?

No. Arithmetic sequences can be written recursively, but recursive notation can describe many other patterns.

How do I check terms generated from a recursive rule?

Substitute each term back into the rule one step at a time and confirm that all initial conditions were used correctly.
